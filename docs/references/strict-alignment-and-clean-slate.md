# 使用者執行偏好：對齊與重置 (Strict Alignment)

根據與使用者的對話反饋，整理出以下關鍵的溝通與執行準則。

## 1. 「修正優先，解釋隨後」
- **痛點**：當代理人發生邏輯錯誤（如跳針、失憶）時，使用者極度厭惡「只有解釋而不見即時修正」的回覆。
- **準則**：一旦發現邏輯失誤（例如邊界設定錯誤），應**立即終止錯誤進程、修正參數並重啟**，然後在回覆中優先報告「我已修正了 X」，而非「這是因為 Y 所以才錯」。

## 2. 「清空重啟」優於「增量清理」
- **場景**：當 LINE 發生訊息同步延遲（通常因多個分頁或記憶體不足造成）時。
- **偏好**：使用者傾向於 **「關閉所有分頁並重新打開唯一一個」** 的徹底重置方案。
- **注意**：執行此動作時需確保 Session (Cookies) 持久化，並在重啟後立即導航回原目標對話。

## 3. 嚴格目標守備
- **準則**：除非使用者明確下達「改訂」指令，否則代理人不得擅自更改原始任務目標（例如擅自加入致歉辭或備選日期）。代理人應展現判斷力，若目標無法達成，應進入守候模式而非亂寫劇本。
